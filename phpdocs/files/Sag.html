login(string $user, string $pass, string $type = null) : mixed

Updates the login credentials in Sag that will be used for all further communications. Pass null to both $user and $pass to turn off authentication, as Sag does support blank usernames and passwords - only one of them has to be set for packets to be sent with authentication.

Cookie authentication will cause a call to the server to establish the session, and will throw an exception if the credentials weren't valid.
Parameters
NameTypeDescription
$userstring

The username you want to login with. (null for none)

$passstring

The password you want to login with. (null for none)

$typestring

The type of login system being used. Defaults to Sag::$AUTH_BASIC.

setDatabase(string $db, bool $createIfNotFound = false) : \Sag

Sets which database Sag is going to send all of its database related communications to (ex., dealing with documents).

When specifying that the database should be created if it doesn't already exists, this will cause an HTTP GET to be sent to /dbName and createDatabase($db) if a 404 response is returned. So, only turn it on if it makes sense for your application, because it could cause needless HTTP GET calls.
Parameters
NameTypeDescription
$dbstring

The database's name, as you'd put in the URL. This string will be URL encoded using PHP's urlencode().

$createIfNotFoundbool

Whether to try and create the specified database if it doesn't exist yet (checks every time this is called).

TypeDescription
